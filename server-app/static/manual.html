Daily Workflow

1

Create A Connection Profile

Open Administration, create an environment, and add the products that exist in that connection.

  • For WebLogic, provide Admin URL, WebLogic credentials, ORACLE_HOME, DOMAIN_HOME, and SSH details.
  • For OIG/OIM, WebLogic is selected automatically. Provide OIG/OIM ORACLE_HOME, DOMAIN_HOME, WebLogic credentials, and xelsysadm credentials.
  • For OUD, provide OUD ORACLE_HOME, OUD instances, and Directory Manager credentials. WebLogic details are optional.
  • For OID, provide OID ORACLE_HOME, cn=orcladmin credentials, and optional WebLogic details.
  • For OAA, provide Kubernetes host SSH, namespace, release name, ingress namespace, runtime URL, and runtime API credentials.
2

Bootstrap And Collect

Use Create And Bootstrap for new profiles. After bootstrap, the dashboard uses the installed runtime SSH key for scheduled collection.

  • Use Test SSH before saving when credentials are new.
  • Use Run Jobs Now from Environment Details when you need an immediate refresh.
  • Scheduled collection runs based on the per-environment interval.
3

Review Server Metrics

Server Metrics shows host reachability, CPU, memory, disk, uptime, WebLogic JVM processes, and cluster node views.

  • For clusters, node tabs show per-host infrastructure details.
  • Health Map gives a quick visual summary before reviewing tables.
  • Hover over warning or down status pills to see the reason.
4

Review Application Metrics

Application Metrics changes based on the products in the profile.

  • ORACLE_HOME tabs show OPatch inventory, patch recommendations, certificates, and product homes.
  • WebLogic Server tabs show domain servers, runtime status, JTA, JVM, sockets, and selected runtime groups.
  • OUD and OID tabs show directory instance data, monitoring, schema, policies, plugin details, and certificates.
  • OAA tabs show pods, Kubernetes resources, Helm version/history, deployment details, certificates, and runtime properties.
5

Use OPatch Recommendations

The dashboard compares installed patches against the bundled KA754 patch catalog for 12.2.1.4 and 14c.

  • Current Stack Bundle shows the installed IDM Stack Patch Bundle when present.
  • Recommended Level shows the latest available SPB currently known to the dashboard.
  • Rows marked Update or Install identify patches that should be reviewed.
Patch data is updated in dashboard releases. Use Check For Updates before a customer demo or quarterly patch review.
6

Analyze Logs

Open Log Analyzer, choose a server or product log source, then select a log file.

  • Double-click a file or click Open to load the complete log.
  • Use Tail for a faster latest-lines view.
  • Use filters for text, severity, wrapping, newest first, and Java exception / stack analysis.
  • Use Detailed Log Analysis for loggers, levels, problem indicators, and searchable parsed entries.
7

Upgrade The Dashboard

Open Support Center from Administration to check version, service paths, health endpoint, GitHub proxy, and upgrade controls.

  • Use Check For Updates from the top bar.
  • Configure proxy settings in Support Center if GitHub access requires a proxy.
  • After upgrade, refresh the browser and confirm the version badge.
8

Get Help

Use the Help / Feedback page when you need assistance with access, collection, metrics, logs, or upgrade issues.

  • Include environment name, product, host, log path, dashboard version, screenshot, and exact error text.
  • Include whether the issue happened during Bootstrap, Run Jobs Now, Test SSH, Tail, Load Complete, or Detailed Log Analysis.
